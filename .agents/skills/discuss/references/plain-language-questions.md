# 비개발자 친화 질문 — Discuss 뱅크

턴당 **질문 1개**. 아래에서 **한 줄**만 골라 쓰고, 괄호 `{…}`를 맥락에 맞게 채운다. (discuss는 옵션 제공 시 `AskQuestion`/`question` 도구(병용)로 묻는다.)

> **opencode CLI**: `question` 도구가 UI 카드로 나타나지 않으면, 채팅에 옵션 불릿 + `(권장)` + 「카드로 고르거나 A/B/C로 답해 주셔도 됩니다」를 쓰고 사용자 텍스트 답을 기다린다. `pending_ask` frontmatter 갱신도 필수.

**옵션 표시 순서 (MUST)**: 2단계 `(권장)` 선정 후 [SKILL.md](../SKILL.md) §**옵션 표시 순서** — `(권장)` **1번**, **마무리 마지막**. 아래 표의 「순서」·「첫·두 번째 옵션」은 **내용 역할**이며, `(권장)`이 아닌 항목은 표시 슬롯이 **고정되지 않음**.

**입력 안내 (매 AskQuestion(`question` 병용) 턴)**: 채팅에 「카드로 고르거나, **A/B/옵션 이름**으로 답해 주셔도 됩니다」 한 줄 포함. 사용자 타이핑은 [turn-decision-tree.md](turn-decision-tree.md) §채팅 텍스트 답변 수용과 동일하게 처리.

## 외부 조사 게이트 (`scan`·`direction` — 턴1 전용)

본 방향·분기 선택 **전** 턴이다. **이 턴에는 방향 후보·(권장) 없음** — 조사 여부만 묻는다.

| 옵션 라벨 (템플릿) | 조사 «예» 시 `(권장)` | 기본 `(권장)` |
| :--- | :---: | :---: |
| **지금 외부·업계 조사 필요** | ✓ | — |
| **route·프로젝트 맥락만으로 진행** | — | ✓ |

- 표시: `(권장)` 옵션을 **1번**, 비권장을 2번에 둔다.

- 「이번 분기({주제})를 정하기 전에, **업계·규제·경쟁 사례 조사**를 먼저 할까요?」→ 위 옵션으로 `AskQuestion`/`question`(병용) 호출.
- **주의**: 게이트 턴에 방향 A/B/C나 `(권장)`을 넣지 않는다 — 본 선택지는 **항상 다음 턴**.
- **스킵**: 사용자가 화면·경로·DISCUSS·증상+영역으로 **이미 좁혔으면** 게이트 생략 → [SKILL.md](../SKILL.md) §조사 게이트 스킵 → 턴2 본 AskQuestion 직행.

## 방향 좁히기 (현황·범위)

- 「지금 가장 답답한 순간은, {진료 중 기록} 할 때인가요, {퇴근 후 정리} 할 때인가요?」
- 「이 부분이 없으면 {진료의}가 가장 먼저 겪는 불편은 무엇인가요? (한 가지만)」
- 「이번에 개선했다고 말할 때, 어디까지 되어 있으면 될까요?」
- 「잘 됐다고 말할 수 있는 기준은, {시간이 ○분 줄었다} / {실수가 줄었다} / {다른 일을 덜 한다} 중 어디에 가깝나요?」
- 「이번에 꼭 손볼 것과, 나중에 해도 되는 것을 한 줄씩만 적어 주실 수 있을까요?」

## 대안 비교

- 「{A: 진료 화면 안에서만 고친다} vs {B: 별도 목록 화면에서 고친다} — staff 입장에서 어느 쪽이 더 자연스러울까요?」
- 「인터넷이 끊겨도 **방금 입력한 내용이 남아야** 하나요, **다른 컴퓨터와 바로 맞춰야** 하나요?」
- 「비슷한 기능을 쓰는 다른 프로그램이 있다면, 이름이나 '○○처럼' 한 가지만 알려 주실 수 있을까요?」

## 엣지 케이스·예외 발굴 (선제 — 사용자 미요청 시에도)

사용자가 «예외도 생각해줘»라고 **말하지 않아도**, `direction`·`converge`·same-session plan **직전**에 **1턴 이상** 쓴다. 질문은 **staff가 겪는 구체 상황**으로 쓰고, 기술 용어·경로는 넣지 않는다.

| 옵션 라벨 (템플릿) | `(권장)` | 용도 |
| :--- | :---: | :--- |
| **{예: 목록이 비었을 때}도 이번에 맞춘다** | 맥락상 가장 빈번 → **표시 1번** | 인범위 → DISCUSS §3·PLAN Edge Case Trace |
| **{예: 저장이 안 됐을 때}도 이번에 맞춘다** | — | 인범위 |
| **둘 다 이번 범위에 넣는다** | — | 인범위 |
| **이번엔 기본 흐름만 — 예외는 나중에** | — | 범위 밖 — §3 「안 하는 것」·Trace `범위 밖` |

**질문 예** (한 턴에 **하나만** 골라 `{…}` 채움):

- 「{기능}을 쓸 때, **목록이 하나도 없을 때** 화면은 어떻게 보여야 할까요? — **빈 안내 문구** / **다른 화면으로 안내** / **이번엔 신경 안 씀**」
- 「**저장이 실패했을 때** staff에게 어떻게 알려 주는 게 좋을까요? — **잠깐 메시지** / **화면에 남는 글씨** / **이번엔 기본 흐름만**」
- 「**인터넷이 끊겨도** 방금 입력한 내용이 남아야 하나요, **다른 컴퓨터와 바로 맞춰야** 하나요, **이번엔 신경 안 씀**?」
- 「**권한이 없는 사람**이 이 화면에 들어오면 — **막기** / **읽기만** / **이번엔 신경 안 씀**?」

**기록**: 답을 DISCUSS §3 **「엣지 케이스」** 불릿 또는 §2 `[확정]`에 1줄씩. same-session plan 시 PLAN `## ⚠️ Edge Case Trace`에 **그대로 이관**.

**converge 전제**: «방향 확정 → 계획으로» 옵션을 넣기 **전**, 위 유형 AskQuestion을 **최소 1턴** 거쳤거나 §3에 **「엣지: 해당 없음 — {사유}」**가 `[확정]`으로 있어야 한다.

## 기존 계획·설계 맹점 심문

- 「이 계획에서 가장 먼저 막힐 것 같은 부분은 {A} / {B} 중 어디일까요?」
- 「잘못 입력했을 때, **바로 고칠 수 있어야** 하나요, **나중에 승인·취소**가 필요한가요?」
- 「사람이 많이 쓸 때 가장 걱정되는 상황은, {동시에 여러 명이 고침} / {느려짐} / {잘못된 정보가 보임} 중 어디인가요?」

## 점검 (리스크·마무리)

- 「출시 전에 **반드시** 한 번은 직접 확인해 보고 싶은 시나리오가 있다면, 한 가지만 말씀해 주실 수 있을까요?」
- 「저장이 됐을 때, staff에게 **어떻게 알려 주는 것**이 좋을까요? (잠깐 메시지 / 화면에 글씨 / 소리 없음)」

## 방향 수렴 (pre-close — `AskQuestion`/`question`(병용) 필수)

수렴 턴에서는 아래 **옵션 라벨**을 `AskQuestion`/`question`(병용)에 넣고, 본문에는 **«이번 discuss에서 끝까지: …»** §1 인용 + 합의 요약만 쓴다. 「정리라고 해 주세요」만 던지지 않는다.

| 옵션 라벨 (템플릿) | `(권장)` | 용도 |
| :--- | :---: | :--- |
| **{예: 예외·오프라인·프린트} 더 논의하기** | **기본** | [close-handoff.md](close-handoff.md) §방향 수렴 — 누락 심층 주제 **발굴** 후 `direction` |
| **방향 확정 → 계획으로** | Ambiguity-Zero **7/7 YES**일 때만 | `close` + **same-session plan 직행** — **메뉴 A·Blueprint 재질문 생략** |
| **{§2 [열림] 문구}만 더 맞추기** | — | min 건수·화이트리스트 등 **한 가지**만 조정 |
| **아직 더 논의** | — | `direction` 계속 |

- **금지**: `[열림]` 0개·추상적 방향만 합의된 상태에서 **방향 확정 → 계획으로**를 `(권장)`로 두기.
- **역할 — 심층 논의**: `{예외·연동·UI 세부} 더 논의하기` — Ambiguity-Zero·엣지 미완이면 **기본 `(권장)`** → 표시 **1번**.
- **역할 — 계획으로**: «방향 확정 → 계획으로» — §3 Ambiguity-Zero **7/7 YES**일 때만 **포함**( `(권장)` **아님**; anti-rush).
- **역할 — 한 항목 조정**: `{§2 [열림] 문구}만 더 맞추기` — 선택 시 **다음 턴**에 그 항목 질문 1개.
- **역할 — 계속 논의**: «아직 더 논의» — `direction` 유지.

## 표준 메뉴 (close·산출물·재진입 — `AskQuestion`/`question`(병용) 필수)

채팅·노트에 「`/plan` 하세요」「`@DISCUSS_…` 붙이세요」만 쓰지 않는다. **assess·분석 옵션 금지**. **마무리는 항상 마지막**.

### 메뉴 A — 설계 전 (close 핸드오프)

**옵션 3개** — `(권장)` **1개**, **마무리 마지막**. `(권장)`·마무리 슬롯 고정, 나머지 1개는 Blueprint vs 더 논의.

| 옵션 (역할) | `(권장)` 조건 |
| :--- | :---: |
| **실행 계획(Blueprint)으로 만들기** | §1·close «계획 문서가 목표?»에 **계획 의도 있을 때** |
| **이 주제 더 논의하기** | 계획 의도 **없을 때** (「끝」이어도 기본) |
| **여기서 마무리** | (항상 **마지막** 슬롯; `(권장)` 아님) |

**표시 순서 예**:

| `(권장)` | 1번 | 2번 | 마지막 |
| :--- | :--- | :--- | :--- |
| Blueprint | Blueprint `(권장)` | 더 논의 | 마무리 |
| 더 논의 | 더 논의 `(권장)` | Blueprint | 마무리 |

- **메뉴 A `(권장)` 조건**: §1 «이번 discuss에서 끝까지: …»·**«계획 문서가 목표?»** 답변에 계획·설계 넘기기 의도가 **있을 때만** Blueprint `(권장)`. **없으면** **이 주제 더 논의하기** `(권장)`, Blueprint `(권장)` **금지**.
- **메뉴 A 전제**: **텍스트** close만 — 「정리/끝/방향 정해졌다」(converge «계획» 선택 **후에는 메뉴 A 금지**).
- **텍스트 close 게이트**: 메뉴 A **전** `AskQuestion`/`question`(병용) 1개 — 「이번 discuss의 목표가 **실행 계획(Blueprint) 문서**까지인가요?」(converge «계획» 경로는 **생략**).
- **converge «계획» 후**: 「방향과 범위는 맞춰 두었습니다» → **PLAN 작성**(same-session) → 산출물 **메뉴 B** — Blueprint **다시 묻지 않음**.
- **금지 (direction/converge)**: §3만 채웠을 때 Blueprint `(권장)` — close 트리거 없이 메뉴 A.
- 노트 §4: Blueprint 선택·PLAN 준비 **후** `핸드오프: plan — YYYY-MM-DD`만.

### 메뉴 B — 설계 후 (산출물 요약·handed-off 재진입)

**3옵션 고정** — `(권장)`=**이 PLAN 전체 순차 실행** → 표시 **1번**; **마무리 마지막**. **「Task 1.1만 먼저 시작」금지**.

| 표시 | 옵션 라벨 (템플릿) | `(권장)` |
| :---: | :--- | :---: |
| 1 | **이 PLAN 전체 순차 실행** | ✓ |
| 2 | **새 주제로 discuss 더 하기** | |
| 3 | **여기서 마무리** | (마지막) |

- **같은 DISCUSS**에 메뉴 A·Blueprint 재안내 **금지**.
- **금지 마감** (AskQuestion/`question`(병용) 없이): 「논의 정본: … 다음에 `/plan`으로 붙여 주세요」

## 용어 치환 (질문에 쓸 때)

| 문서·개발 용어 | 질문에서 쓸 말 |
| :--- | :--- |
| Must / Should | 이번에 꼭 넣는 것 / 있으면 좋은 것 |
| AC, Given/When/Then | 확인 방법, "이렇게 하면 통과" |
| Primary user | 가장 많이 쓸 사람 (역할 이름) |
| Scope in/out | 이번에 하는 것 / 하지 않는 것 |
| Widget, panel, API | 화면·목록·저장 (필요할 때만) |
| Offline / sync | 끊겨도 남음 / 다른 곳과 바로 같음 |
